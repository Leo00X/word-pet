# 文档同步更新工作流

## 使用场景
**代码审查通过后**，同步更新项目文档记录新功能或修复的问题

## ⚠️ 注意
- 先完成 `/3-qa-code-review` 确保代码质量
- 然后再使用本工作流更新文档
- 最后使用 `/5-git-commit` 提交

---

## 📋 更新清单

### Step 1: 更新 FEATURES.md

**位置**: 在对应功能类别下添加

```markdown
## 🎮 成长与互动

### 5. 新功能名称 ⭐⭐⭐⭐⭐

**状态**: ✅ 已完成 (2025-12-06)

**功能描述**:
- 功能点 1
- 功能点 2
- 功能点 3

**技术实现**:
- Composable: `useXxx.js`
- UI: `XxxPanel.vue`

**使用方式**:
```
主页 → 某某 Tab → 功能入口
```
```

---

### Step 2: 更新 README.md

#### 2.1 更新开发进度（Roadmap）

找到 `## ✅ 开发进度` 区域，添加：

```markdown
[x] **新功能名称** (功能说明) ✅ 2025-12-06
```

#### 2.2 更新最新动态（可选）

```markdown
> **📅 最新更新**: 2025-12-06 - 🎉 新功能名称完成
```

---

### Step 3: 更新 docs/maintenance/UPDATES.md（按需）

**使用场景**: 重大功能发布或版本更新时

**注意**:
- ✅ 简要记录功能发布和新增内容
- ✅ Bug 修复只需概括（详细记录在 BUG.md）
- ❌ 不要记录详细的修复过程

**模板**:
```markdown
## 🎉 2025-12-06 - 功能名称

### ✨ 新增功能

**功能名称**:
- [功能点 1]
- [功能点 2]

**相关模块**:
- `useXxx.js` - [功能说明]  
- `XxxPanel.vue` - [UI 组件]

### 🐛 Bug 修复

修复了 X 个相关问题：
- 问题概述 1
- 问题概述 2

详细修复记录请查阅 [BUG.md](./BUG.md)
```

---

### Step 4: 检查 DEVELOPMENT.md（按需）

如果新增了通用的 Composable 或重要模块：

**位置**: `## 核心模块详解`

```markdown
#### 26. useXxx.js - 功能说明

**主要功能**:
- [功能 1]
- [功能 2]

**导出内容**:
```javascript
{
  data,          // 状态
  loadData,      // 方法
  saveData
}
```
```

---

### Step 5: 检查 docs/guides（按需）

如果新增了**面向用户的功能**（需要用户使用说明）：

**位置**: `docs/guides/XXX_GUIDE.md`

**适用场景**:
- 新的游戏玩法
- 新的系统（如商城、日记）
- 需要配置说明的功能

**模板**:
```markdown
# [功能名称] 使用指南

## 功能介绍

[简要说明功能用途]

## 使用方法

### 1. 打开功能
[步骤说明]

### 2. 核心操作
[功能操作说明]

## 常见问题

### Q1: [问题]
A: [答案]
```

**提示**: 用户指南应该**简洁易懂**，面向非技术用户

---

### Step 6: 检查 docs/maintenance/BUG.md（如有）

如果修复了已知 Bug，更新状态：

```markdown
## [已修复] BUG #XX - 问题描述

**修复时间**: 2025-12-06
**修复方案**: [简要说明]
```

---

## 📖 文档风格指南

### Markdown 格式规范

**1. 标题层级**
```markdown
## 二级标题 - 功能类别
### 三级标题 - 具体功能
#### 四级标题 - 细节说明
```

**2. 代码块**
```markdown
```javascript
// 代码示例
```
\```
```

**3. 列表**
```markdown
- 无序列表：功能点
1. 有序列表：步骤

- [ ] 未完成任务
- [x] 已完成任务
```

**4. 链接**
```markdown
[文档名称](./path/to/doc.md)
[章节名称](./doc.md#章节)
```

**5. 表情符号**
```markdown
✅ 完成
⚠️ 警告  
🎉 新功能
🐛 Bug 修复
📝 文档更新
```

---

## ⚠️ 注意事项

### README 保护协议

**禁止**:
- ❌ 重写整个 README
- ❌ 修改项目介绍部分
- ❌ 删除现有内容

**允许**:
- ✅ 在 Roadmap 添加新条目
- ✅ 在文档导航添加链接
- ✅ 更新最新动态

### 文档一致性

更新文档时确保：
- [ ] 日期一致（所有文档使用同一日期）
- [ ] 功能名称一致
- [ ] 状态标记一致（✅ 或 ⚠️）

---

## 📋 更新检查清单

完成以下至少 3 项：

- [ ] `FEATURES.md` - 添加功能说明
- [ ] `README.md` Roadmap - 标记完成
- [ ] `UPDATES.md` - 添加更新日志
- [ ] `DEVELOPMENT.md` - 添加模块说明（可选）
- [ ] `BUG.md` - 标记 Bug 修复（如有）
